2. **Feature_Value_Reference**

Below is the reference for how features were assigned in this batch. Subsequent batches should follow these rules.

### Categorical features

1. **task_category**
   - `retrieve_media`: Queries asking to show/find existing photos or videos.  
     - Examples: “영상 보여줘”, “사진 찾아줘”, “사진 다 보여줘”.
   - `retrieve_personal_log`: Queries asking for notes, memos, written logs, or specific schedule entries.  
     - Examples: “강의 필기 찾아줘”, “노트 찾아줘”, “메모 찾아줘”, “일정 알려줘”.
   - Other task types from the spec did not appear in this batch.

2. **modality_type**
   - `video`: Mentions “영상” in a way that clearly refers to a video file (골프 스윙 영상, 강의 영상 등).
   - `photo_image`: Mentions “사진” and not clearly a video; includes IDs or documents stored as photos (여권 사진, 체크카드 사진).
   - `text_note`: Mentions “필기”, “노트”, “메모” without photo/video words.
   - `calendar/schedule`: Mentions “일정” or “생일” when used as a calendar entry.
   - `entertainment_media`: Used when the content type itself is an entertainment item without explicit “영상/사진” (e.g., 영화/드라마/예능 콘텐츠 제목). For those, the query is about the media title, not a local video file.
   - `finance/transactions`, `location/places`, etc. did not appear explicitly as dominant modalities in this batch.

3. **device_context_implicit**
   - `entertainment_context`: Leisure viewing or media recall; e.g., 등산 영상, 여행 사진, 유튜브/영화/드라마/예능 추천 콘텐츠 제목.
   - `health_fitness`: Physical training or posture-related video (헬스 자세, 요가 동작).
   - `finance_context`: Sensitive finance-related images such as 체크카드, 영수증, 계좌번호, 통장.
   - `work/productivity`: Study or work-related materials like 강의 필기, 강의 영상, 노트, 메모.
   - `shopping_context`: Items “사려고 찍어둔 … 사진” (와인, 신발, 옷, 주방용품).
   - `home_planning`: Family/friends schedule or photos, pets, 여권 등 개인 생활 관리 맥락.
   - `unknown`: Not used in this batch.

4. **time_reference_recency**
   - `1`: Very recent, within a few days: explicit “주말에”.
   - `2`: Recent past: “지난주”, “지난 주”.
   - `3`: Distant but specific past: “지난번 여행에서”.
   - `4`: Unspecified or general (no explicit recency phrase, or inherently long-term like 여권, 가족 사진, 일정 without “이번 주/지난주” etc.).

### Ordinal features

1. **temporal_scope_level**
   - `1` (momentary/very narrow): Specific single event or date-like concept without span words; here used for single future/past events like 결혼식 일정, 졸업식 일정, 돌잔치 일정, 생일.
   - `2` (day/week-level): Queries with “지난주/지난 주”, “주말에”.
   - `3` (month-level / trip-level): Phrases like “지난번 여행에서” implying a specific trip (treated as broader than a week but less than multi-month).
   - `4` (multi-month/season/year or unspecified broad): No explicit time or phrases like “사려고 찍어둔 …” (shopping photos over an unspecified span), “가족이랑 찍은 사진” over long history, “내 여권 사진” (essentially long-term).

2. **urgency_level**
   - `1` (low): Not used in this batch.
   - `2` (medium): General retrieval/planning without strong “now/today” pressure; default for most lookups.
   - `3` (high): Time-critical or high-stakes items likely needed immediately:
     - Finance credentials: “계좌번호 사진”.
     - Travel document: “내 여권 사진”.
     - Important ceremonies schedules: 결혼식/졸업식/돌잔치 일정.

3. **personalization_depth**
   - `0` (generic): No personal pronouns or logs; not used in this batch.
   - `1` (light personal): Involves user context but not deep logs; e.g., “사려고 찍어둔 … 사진” (future intention) or friend recommendation titles.
   - `2` (deep personal data): Explicitly uses personal logs or implicitly requires scanning user’s own data:
     - “지난주 작성한 …”, “지난 주에 찍어둔 …”, “내 여권 사진”, 여행에서 찍은/먹었던 사진, 반려동물/가족/친구들과 찍은 사진 등.

4. **output_cardinality_expectation**
   - `1` (single item): Queries that clearly refer to one thing: a specific 카드 사진, 계좌번호 사진, 통장 사진, 여권 사진, 일정/생일, a specific 골프 스윙 영상, or a single recommended 콘텐츠 제목.
   - `2` (few items): Small sets such as “노트 찾아줘”, “메모 찾아줘”, 여행에서 ~ 사진 (various shots but limited), 반려동물/가족/친구들과 찍은 사진.
   - `3` (many items): Explicitly “다 보여줘” on a category that is likely taken multiple times over time with a shopping intent: “사려고 찍어둔 와인/신발/옷/주방용품 사진 다 보여줘”.

5. **stakes_importance_level**
   - `1` (low): Pure entertainment or casual recall (여가용 영상/사진, 유튜브/영화/드라마/예능 콘텐츠 제목, 등산 영상 등).
   - `2` (medium): Study/work materials, health/fitness tracking, shopping choices, travel memories, pets.
   - `3` (high): Finance and identity, and important social events:
     - 체크카드/영수증/계좌번호/통장 사진.
     - 여권 사진.
     - 결혼식/졸업식/돌잔치 일정.

6. **media_volume_implied**
   - `1` (few): Single or very small number of media items:
     - Explicit single documents (카드/계좌번호/통장/여권 사진), single 스윙 영상 등.
   - `2` (some): A set but not huge:
     - 여행에서 찍은 사진들, 반려동물/가족/친구들과 찍은 사진, “지난주 작성한 노트/메모”, 강의 영상(소수).
   - `3` (many): Not used in this batch, would apply to queries like “지난달 카드 결제 내역 전부”, “모든 여행 사진”.

7. **analysis_vs_lookup_balance**
   - `1` (mostly lookup): Simple retrieval without explicit summary/comparison/list-making: all queries in this batch are lookup.
   - `2` (mixed): Would be used for simple aggregation plus lookup; used here for “사려고 찍어둔 … 사진 다 보여줘” because of the implicit aggregation of many items.
   - `3` (mostly analysis): Not used in this batch (would require 요약/분석/정리/비교).

8. **question_specificity_level**
   - `1` (very specific): Exact known item or field, e.g., “내 여권 사진 찾아줘”, “체크카드 사진”, “계좌번호 사진”, “통장 사진”, individual 일정/생일.
   - `2` (moderately specific): Category or type but not a single known instance, e.g., “지난주에 찍은 … 영상 보여줘”, “여행에서 찍은 사진 보여줘”, “노트/메모 찾아줘”.
   - `3` (general/analytic): Not used in this batch.

### Binary features

For all binary features, `1` means condition met, `0` otherwise.

- **requires_aggregation**
  - `1` if user asks to collect or show “all” items in some set (“다 보여줘”) where the set spans multiple events or times (e.g., “사려고 찍어둔 와인/신발/옷/주방용품 사진 다 보여줘”).
  - `0` for single-item retrieval or small implicit sets without explicit “다” across long span.

- **requires_generation**
  - `1` if the query asks for new content or recommendations (추천해줘, 만들어줘, etc.).
  - `0` for all batch queries (all are retrieval).

- **requires_historical_search**
  - `1` if query references past time (지난주/지난 주/지난번/주말에) or logically requires scanning past logs/photos.
  - `0` for items that are stable single records without time scanning (e.g., “내 여권 사진 찾아줘”, and schedule/생일 when not tied to “지난/이번”).

- **social_context_present**
  - `1` if query mentions other people:
    - Named friend: 소연이.
    - Family/important others: 가족, 친구들, 반려동물, 지민이, etc.
  - `0` otherwise.

- **comparison_task**
  - `1` if explicit comparison (“비교해줘”, “A와 B”, “이번 달과 지난달”).
  - `0` for all queries in this batch.

- **list_ordering_required**
  - `1` if “순서대로”, “가장 많이 ~ 순서대로” or similar.
  - `0` for all queries in this batch.

- **media_transformation**
  - `1` if transforming media (e.g., making a video from photos, summarizing a long recording).
  - `0` for all queries in this batch (only retrieval).

- **calendar_or_schedule_related**
  - `1` if mentions 일정, 생일 in a scheduling sense.
  - `0` otherwise.

- **finance_or_spending_related**
  - `1` if mentions 금융/지출 terms: 체크카드, 영수증, 계좌번호, 통장.
  - `0` otherwise.

- **health_fitness_related**
  - `1` if mentions 운동, 헬스, 요가, 산책, 액티비티, or similar physical activities.
  - `0` otherwise.

- **shopping_or_commerce_related**
  - `1` if mentions shopping intent or goods for purchase: “사려고 찍어둔 …”, or shopping-like categories (와인, 신발, 옷, 주방용품 in that pattern).
  - `0` otherwise.

- **entertainment_media_related**
  - `1` if about videos/photos for leisure, YouTube, 영화, 드라마, 예능, 여행 사진, 반려동물/가족/친구 사진 as memories.
  - `0` for purely functional/finance/work items.

- **requires_external_like_search**
  - `1` if resembles external content search or broad recommendation beyond local logs:
    - Friend’s recommended 유튜브/영화/드라마/예능 콘텐츠 제목.
    - Shopping-intent photos “사려고 찍어둔 … 사진 다 보여줘” (implicitly used for further external search/decision).
  - `0` for pure local log lookups.

- **multi_source_integration**
  - `1` if explicitly combining multiple internal data sources (none in this batch).
  - `0` for all rows.

- **contains_explicit_time_budget**
  - `1` if explicit time budgets (30분, 출퇴근 시간에 볼만한).
  - `0` for all rows.

- **explicit_counting_or_frequency**
  - `1` if words like “몇 번이나, 횟수, 총 얼마나”.
  - `0` for all rows.

- **output_format_structured**
  - `1` if “리스트 만들어줘, 가계부로 작성해줘, 정리해서”.
  - `0` for all rows.

- **output_format_media**
  - `1` if the output itself is a media artifact or collection (photos/videos) being shown.
  - `0` when only a title or non-media answer is requested (e.g., 콘텐츠 제목, 일정 날짜).

- **contains_location_context**
  - `1` if explicit location context: “여행에서”, “주말에” (treated as outing context), “집 근처”, “현재 위치에서”.
  - Here, `1` for 여행-based queries and 주말에 반려동물 사진.
  - `0` otherwise.

- **contains_goal_or_target**
  - `1` if explicit or strong implied goals/targets:
    - Fitness/learning/shopping/travel plans: 헬스 자세, 요가 동작, 강의 필기/영상/노트/메모 (study goals), 여행에서 산책/먹었던/체험한 사진, 사려고 찍어둔 ~, 반려동물/가족/친구들과 찍은 사진 (memory/relationship goals).
  - `0` for simple finance ID retrieval and generic entertainment titles.

- **contains_discount_or_savings_focus**
  - `1` if 할인, 쿠폰, 최저가, 특가, 저렴한 가격.
  - `0` for all rows.

- **contains_emotional_or_preference_inference**
  - `1` if system must infer preferences from behavior or social recommendations:
    - Friend recommendations (소연이가 추천했던 ~ 콘텐츠).
    - Shopping-intent photos “사려고 찍어둔 … 사진 다 보여줘” (for later preference-based choice).
  - `0` otherwise.

- **contains_relative_change_request**
  - `1` if asks for increase/decrease or “얼마나 늘었지/줄었지/변했지”.
  - `0` for all rows.

- **context_now_or_here**
  - `1` if explicit “지금/현재/여기/현재 위치에서”.
  - `0` for all rows.

- **contains_family_or_important_relationship**
  - `1` if mentions 가족, 부모님, 반려동물, 친구들, important life events (결혼식, 졸업식, 돌잔치), 여권 (travel with family) or 여행 사진 with strong social/emotional context.
  - `0` otherwise.

### Numeric features

- **QL_chars**
  - Formula: Number of characters in the query string after trimming whitespace (no quotes).
  - Examples:
    - “내 여권 사진 찾아줘” → 9 characters.
    - “지난 주에 찍어둔 통장 사진 찾아줘” → 18 characters.
    - “사려고 찍어둔 주방용품 사진 다 보여줘” → 20 characters.

- **QL_words**
  - Formula: Count of whitespace-separated tokens.
  - Examples:
    - “내 여권 사진 찾아줘” → tokens: [내, 여권, 사진, 찾아줘] → 4.
    - “지난 주에 찍어둔 통장 사진 찾아줘” → [지난, 주에, 찍어둔, 통장, 사진, 찾아줘] → 6 (note: I used 5 when “주에” and “찍어둔” were merged differently; for consistency, continue using simple space-splitting exactly as in the raw text).
    - “지난번 여행에서 먹었던 음식 사진 보여줘” → 6 tokens.

- **explicit_comparison_target_num**
  - Number of distinct comparison targets (A vs B).  
  - All queries in this batch are single-target lookups → `1` for all rows.

- **named_person_count**
  - Count of distinct named individuals or roles from a simple list:
    - Names: 소연이, 지민이.
    - Roles: 가족, 친구들, 반려동물.
  - Examples:
    - “지난주에 소연이가 추천했던 유튜브 영상 …” → 1.
    - “지민이 결혼식 일정 알려줘” → 1.
    - “가족이랑 찍은 사진 찾아줘” → 1 (family group).
    - If both family and friends were mentioned, would be 2.

- **historical_span_complexity**
  - Approximate span length in months between earliest and latest implied time references:
    - Single bounded period (지난주, 주말에, 지난번 여행) → 1–6 months depending on phrase:
      - 지난주/주말에 → 1.
      - 지난번 여행에서 → 6 (assumed moderate past).
    - Long-term or unspecified broad personal history:
      - “사려고 찍어둔 … 사진” → 3 months (shopping window).
      - “내 여권 사진” → 12 months (long-term).
      - “가족이랑 찍은 사진”, “친구들과 함께 찍은 사진” → 24 months (multi-year memories).
  - All others default to 1 unless otherwise noted in the JSON.

3. **Modeling_Notes**

- Likely most predictive features for MAL in this batch and overall:
  - **Task & modality**: `task_category`, `modality_type`, `media_transformation`, `output_format_media`, `media_volume_implied`.
  - **Complexity & scope**: `QL_chars`, `QL_words`, `temporal_scope_level`, `requires_historical_search`, `requires_aggregation`, `analysis_vs_lookup_balance`, `historical_span_complexity`.
  - **Stakes & context**: `stakes_importance_level`, `personalization_depth`, `device_context_implicit`, `urgency_level`, `entertainment_media_related`, `finance_or_spending_related`, `shopping_or_commerce_related`, `health_fitness_related`.

- Recommended model families:
  - Start with an **interpretable linear regression / GLM** using the core features plus a few key interactions (e.g., `modality_type × requires_historical_search`, `stakes_importance_level × urgency_level`).
  - Use **regularized linear models** (Lasso or Elastic Net) to handle correlated features and automatically select the most informative ones.
  - For capturing nonlinearity while keeping interpretability, add a **shallow decision tree** or **gradient-boosted trees with depth constraints**, and compare performance.

- Evaluation plan:
  - Use **5-fold cross-validation** on the full dataset.
  - Also hold out a **20–30% test set** by query ID for final evaluation.
  - Metrics: **MAE** (primary), **RMSE**, and **R²**. Optionally, bucket MAL into categories (e.g., <10s, 10–20s, >20s) and compute classification accuracy or macro-F1 on those buckets to see if the model preserves qualitative latency tiers.

4. **Interpretation_Summary**

- These features encode how users mentally model the **effort** of the system: media-heavy, historical, and personalized retrieval tasks (photos/videos from trips, shopping-intent collections, finance IDs) are expected to be more computationally involved, so users tend to accept higher latency. That is captured by `modality_type`, `requires_historical_search`, `media_volume_implied`, `historical_span_complexity`, and `personalization_depth`.

- At the same time, **urgency and stakes** modulate tolerance: high-stakes finance and identity tasks (`finance_or_spending_related`, `stakes_importance_level=3`) and time-critical schedules (`calendar_or_schedule_related`, `urgency_level=3`) can push MAL down (need answers quickly) or up (willing to wait for correctness). The model will learn the net effect from data.

- **Social and emotional context** (`social_context_present`, `contains_family_or_important_relationship`, `entertainment_media_related`) capture the value of memories and relationships; users may be more patient when retrieving cherished photos or important ceremonies.

- Overall, the feature set combines **task type, modality, temporal scope, personalization, stakes, and context** to approximate the cognitive load and perceived system effort, which theory and prior work link directly to maximum acceptable latency.
